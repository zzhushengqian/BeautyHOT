# BeautyHOT

把公开网页或 BeautyHOT API 中的美妆行业动态整理成可追溯、可发布的中文情报。优先准确性、来源透明度和行业意义，不追求堆砌数量。

## 先读取参考资料

- 每次检索资讯前读取 [references/sources.md](references/sources.md)，按来源层级搜索和核验。
- 分类、去重、排序或生成网站数据时读取 [references/taxonomy.md](references/taxonomy.md)。
- 接入或建设 BeautyHOT 网站 API 时读取 [references/api-contract.md](references/api-contract.md)。

## 选择数据入口

1. 用户提供 BeautyHOT API 或网站公开接口时，先检查其 OpenAPI、接口文档或返回结构，再按 [references/api-contract.md](references/api-contract.md) 调用。
2. 没有可用 API 时，使用实时网页搜索。优先官方公告、公司新闻稿、交易所文件和监管机构，其次使用可靠通讯社与垂直行业媒体。
3. 遇到登录、订阅、验证码、反爬、付费数据库或 API Key 时停止访问该来源，明确告诉用户所需权限及可替代的公开来源。不得绕过访问控制。

## 理解用户意图

按下列规则确定范围：

- “今天/过去 24 小时”：使用滚动 24 小时时间窗。
- “最近/近期”但未给范围：默认最近 7 天，并在输出中写明。
- “本周/最近一周”：最近 7 天。
- “日报”：生成按自然日或用户指定日期汇总的编辑成品。
- “全部/完整/全量”：扩大召回并保留次要条目；否则只输出精选。
- 指定品牌、人物、赛道或事件类型：同时搜索中文名、英文名、母公司、品牌别名和常见译名。
- 未指定地域：兼顾中国与全球；优先与中国市场、重要集团及高增长品牌有关的事件。

## 工作流

### 1. 制定检索矩阵

将查询拆成“时间 × 地域 × 类别 × 实体”。宽泛查询至少覆盖：

- 品牌与公司
- 人事与组织
- 投融资、并购与上市
- 财报与经营
- 产品、成分与研发
- 渠道与零售
- 营销与消费者趋势
- 法规与合规

针对每类组合中英文关键词。使用 [references/sources.md](references/sources.md) 中的检索模板，避免只搜一个聚合页。

### 2. 记录候选事件

每条候选至少记录：

- 规范化标题
- 事件日期和首次公开时间
- 涉及品牌、公司、人物
- 类别与地域
- 事实摘要
- 原始来源 URL、来源名称、发布日期
- 第二来源（如有）
- 是否为传闻、独家报道或官方确认

没有可追溯 URL 的条目不得进入正式简报。

### 3. 核验关键事实

对人事、投融资、并购、财报、处罚和法规采用更高核验标准：

1. 优先找到公司公告、监管文件、交易所披露或当事人公开声明。
2. 若只有媒体报道，至少找两个相互独立的可靠来源。
3. 金额必须区分融资额、交易对价、估值、企业价值和传闻报价。
4. 人事必须区分任命、晋升、兼任、离任和临时代理，并核对生效日期。
5. 对“拟收购、洽谈中、已签约、已交割”分别表述，不得合并成“完成收购”。
6. 无法确认时标记“尚未获官方确认”，不要补全缺失事实。

摘要是转述，不是原文引用。需要引用原话时回到原始来源核对。

### 4. 去重与聚合

按“同一主体 + 同一事件 + 相近日期”合并转载和跟进报道。保留最接近事实源头的链接，把补充信息放入同一条目。不要把融资传闻、正式宣布和完成交割误合并为同一阶段。

### 5. 编辑排序

按以下因素综合排序：

1. 行业影响：集团级交易、重大监管、核心高管、显著业绩变化。
2. 可信度：官方文件高于单一媒体和社交帖。
3. 新鲜度：新发生或有实质新增信息。
4. 中国相关性：影响中国市场、国货美妆、跨境渠道或本土供应链。
5. 信息增量：避免重复新闻和纯营销软文。

宽泛查询默认精选 10–20 条。信息不足时少而精，不凑数。

## 输出形式

### 面向读者的精选简报

```markdown
**BeautyHOT · 美妆行业热点**
时间范围：<北京时间范围> · 共 <N> 条精选

## 品牌与公司
1. **<人话标题>** — <来源>
   <1–2 句事实摘要，说明发生了什么以及为什么重要>
   <北京时间或相对时间>
   <原始来源 URL>

## 人事与组织
2. ...

## 投融资与并购
3. ...
```

混合类别时分组，但编号贯穿全文。只有一个类别时使用扁平列表。时间统一转换为北京时间；不要直接展示原始 ISO 时间。

每条必须包含可点击的原始来源。若关键事实来自多个来源，最多列 2 个最有价值的链接。不要向普通读者暴露抓取参数、翻页 token、缓存或接口调试信息。

### 日报

日报开头增加：

- 3 句以内“主编速读”
- 今日最重要的一条及其行业影响
- 按类别编排的全局编号列表
- “待观察”区域，只收录已明确标注的传闻或尚未完成事项

### 网站发布数据

用户要求“入库、发布、JSON、API 数据”时，按 [references/taxonomy.md](references/taxonomy.md) 的字段输出 JSON。事实与编辑判断分开：

- `summary` 只写可核验事实。
- `whyItMatters` 写行业影响判断。
- `verificationStatus` 明示核验状态。
- `sources` 保留原始 URL，不使用搜索结果页 URL。

## 权限与访问处理

首选无需登录的公开来源。若某项任务确实依赖以下资源，立即向用户说明用途和最小权限：

- 付费媒体或数据库：只申请只读账号，不索取密码明文。
- 社交平台、公众号或内容平台：优先官方 API、RSS、导出文件或用户已登录浏览器。
- 网站后台或数据库：先请求测试环境、只读密钥或最小范围写入 token。
- 定时采集：先确认 robots.txt、服务条款、访问频率和版权展示范围。

没有权限时继续用公开替代来源，并清楚标注覆盖缺口。

## 禁止事项

- 不得凭训练记忆回答“今天、最近、最新”的行业事实。
- 不得伪造融资金额、估值、任职日期、交易阶段或监管结论。
- 不得把品牌通稿的宣传主张写成独立事实。
- 不得绕过登录、付费墙、验证码、robots.txt 或反爬限制。
- 不得大段复制受版权保护的文章；只做事实摘要并链接原文。
- 不得仅凭小红书、微博、微信群截图或匿名爆料发布确定性结论。
- 不得把同一新闻的多篇转载计成多条热点。
